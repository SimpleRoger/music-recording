However as of late, now it's been returning an error whenever I ask it for audio only or printing titles. It will give me Requested format is not available. Use --list-formats for a list of available formats. However, if I take away --cookies-from-browser, then it works just fine? What??

Below I pasted two outputs. One of me downloading audio with --cookies-from-browser, and one without. You'll see the first one fails telling me my requested format is unavailable by somehow having --cookies-from-browser set.

This also happens when attempting to print title from videos with --cookies-from-browser added. My current work around for printing titles is adding -f sb0 at the end, however I shouldn't need to do that since this wasn't a problem before, and it's still a problem with audio downloading with no solution currently.

No supported JavaScript runtime could be found. Only deno is enabled by default; to use another runtime add --js-runtimes RUNTIME[:PATH] to your command/config. YouTube extraction without a JS runtime has been deprecated, and some formats may be missing. See https://github.com/yt-dlp/yt-dlp/wiki/EJS for details on installing one

You need a JavaScript runtime in order to solve the js challenges presented by YouTube, if you don't have one (supported ones are deno, node, bun, quickjs), install deno by running winget install DenoLand.Deno or check their installation page.

For context: yes, this requirement was added recently due to frequent YouTube changes, without cookies some clients that don't support authentication are requested so you may still get some formats
